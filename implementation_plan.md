# Attendee Module: Medical Records & Prescription Monitoring

This implementation plan outlines the creation of the Medical Records and Prescription Monitoring modules for the Attendee dashboard. These modules will allow an attendee to upload/view/print medical records and track/download doctor prescriptions.

## Proposed Changes

---

### Database Layer
#### [NEW] SQL Table: `medical_records`
Will store patient lab reports, scan reports, etc.
- `id` (INT)
- `patient_code` (VARCHAR) - links to patient
- `record_type` (VARCHAR) - Lab Report, Scan Report, etc.
- `file_path` (VARCHAR) - path to securely uploaded file
- `description` (TEXT)
- `uploaded_at` (TIMESTAMP)

#### [NEW] SQL Table: `prescriptions`
Will store prescriptions uploaded/generated by doctors.
- `id` (INT)
- `patient_code` (VARCHAR)
- `doctor_code` (VARCHAR)
- `file_path` (VARCHAR) 
- `notes` (TEXT)
- `uploaded_at` (TIMESTAMP)

### Attendee Backend & UI (Using Attendee Dashboard Theme)
#### [NEW] [Attendee/medical_records.php](file:///d:/xamp/htdocs/CareSync/Attendee/medical_records.php)
- Glassmorphism UI identical to `attendee_dashboard.php`.
- Provides an **Upload Form** for PDFs and Images (Lab/Scan reports).
- **Patient Report List**: Fetches from `medical_records` to view historical diagnosis reports.
- **Download/Print Logic**: Buttons that allow admins to directly view or download the attached `file_path` from the browser.

#### [NEW] [Attendee/prescriptions.php](file:///d:/xamp/htdocs/CareSync/Attendee/prescriptions.php)
- Glassmorphism UI matching the theme.
- **Monitoring Table**: Queries the `prescriptions` table to show prescriptions written by doctors.
- Includes Patient Name, Doctor Name, Date, and **Download/Print** actions.
- Tracks exactly which appointment the prescription belongs to.
